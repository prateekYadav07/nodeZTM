JS Single Threaded

7. JS is a single threaded language so it requires other components to work asycnhronously.

JS Runtime environment

8. Browser implements a javaScript run time environment which consists of JS engine, Web APIs, Callback Queues and Event loop

Node

9. node is a JS runtime outside of browser which consists of V8 engine for JS and LibUV for Web APIs, callback queue and event loop.

Node REPL

10. Node provides a Read-Eval-Print-Loop environment where one can write js code directly, not preferable though.

Node vs JavaScript

11. Node has certain functionality different than what we see in javaScript.

Node MultiThreading

12. Since js is a single threaded lang, Node add certain functionality to make multithreading easy, For asynch tasks like filesystems i/o it uses thread pool of 4 concurrent independent threads, while network ops are handled by OS kernel, Event loop is responsible for sending back callbacks returns.

Phases of Event Loop

13. Event loop works in phases with mainly 4 types of callback queues shown above. All this phases have their respective callback functions waiting in queue to be executed. One cycle of all these phase completion is called 1 event loop.

Traditional Model

14. Programming langs like Php and python uses a web server like apache or nginx to take care of requests. But it has blocking elements, to hadnle this server spawns a separate thread for every client and that creates an overhead on resources

Node Model

15. In case of Node there's libUV which takes of care of asynchronous operation and event loop which performs callback execution. Also there's a limited set of thread pool hence no need to spawn more threads.
